## Project Notes

**Image Browser - Final CSS Fix for Masonry**

Changed [src/server/template.html](src/server/template.html:10):
```css
/* Before (incorrect) */
.masonry-item img { width: 100%; height: auto; }

/* After (correct) */
.masonry-item img { max-width: 100%; height: auto; }
```

This fix ensures images of any aspect ratio display correctly in the CSS columns masonry layout. Combined with the width/height HTML attributes from the earlier dimension storage implementation, this provides:
1. Correct aspect ratio rendering for all images
2. No layout shift/flashing when images load
3. Responsive behavior across different column counts

All 300 meme images (300x300 squares) now display correctly, and the system will handle mixed aspect ratios correctly if non-square images are added in the future.
